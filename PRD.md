# PRD: AI 기반 마인드맵 자동 생성 웹서비스

## 1) 서비스 오버뷰
사용자가 **주제(또는 목적/대상/제약)** 만 입력하면, AI가 해당 주제를 체계적으로 분해해 **일정 수준 이상의 품질을 갖춘 마인드맵**을 자동으로 생성해 주는 웹서비스.

- 핵심 가치: “자료 조사 + 생각 정리”를 한 화면에서 빠르게 시작하고, **누락을 줄이며**, **제작 시간을 크게 단축**한다.
- 접근성: 웹 기반으로 언제 어디서든 접속 가능.

## 2) 문제 정의
마인드맵을 통해 생각을 정리하려 할 때,
- 자료 조사와 생각 정리를 동시에 수행하면 시간이 오래 걸리고,
- 주제에 대한 사전 지식이 부족하면 중요한 하위 항목이 누락될 수 있다.

따라서 사용자가 별도의 지식 없이도 **체계화된 마인드맵을 자동으로 작성**할 수 있는 서비스가 필요하다.

## 3) 목표(Goals)
- 마인드맵 제작 시간을 낮춰 사용자 생산성을 높인다.
- 사전 지식이 부족한 주제에서도 기본 골격/카테고리/하위 항목을 제안해 **누락을 최소화**한다.
- 누구나 쉽게 시작할 수 있도록 입력/사용 흐름의 마찰을 줄인다.

## 4) 비목표(Non-goals)
- 특정 도메인(예: 법률/의료 등)의 전문 자문 제공을 목표로 하지 않는다.
- 프로젝트 일정/실행 계획(마일스톤, 인력, 일정 산정 등)은 본 PRD 범위에 포함하지 않는다.

## 5) 타겟 사용자 및 핵심 사용 사례
### 타겟 사용자
- 시간을 아끼고 생산성을 높이고 싶은 모든 사용자
- 특히 기획/업무/학습 과정에서 빠르게 구조화를 시작해야 하는 사용자

### 사용 사례(예시)
- 기획 단계에서 보조 도구로 사용하여 기획 기간을 줄이고, 실무에 더 많은 시간과 노력을 할애

## 6) 사용자 니즈 / JTBD
“사전 지식이 부족한 주제에서도, 빠르게 생각의 뼈대를 잡고 누락 없이 정리하기 위해, 자동으로 일정 품질의 마인드맵을 얻고 싶다.”

## 7) 제품 범위(스코프)
### MVP 범위(In scope)
- **입력**: 주제(필수) + 선택 입력(목적/대상/관점/깊이/톤 등)
- **생성**: AI 기반 마인드맵 자동 생성(주요 가지/하위 항목 구조화)
- **편집**: 노드 추가/삭제/이름 변경 등 기본 편집
- **내보내기/공유**: 이미지(예: PNG) 또는 링크 공유(최소 1개 방식)

### 제외 범위(Out of scope)
- 실시간 다중 사용자 협업 편집(향후 고려)
- 고급 리서치 자동화(웹 크롤링 기반 출처 수집 등, 향후 고려)

## 8) 핵심 기능 요구사항(High-level)
1. **빠른 생성 경험**
   - 주제만으로도 생성 시작 가능(저마찰 입력)
   - 생성 결과가 “바로 편집 가능한” 구조로 제공
2. **품질/누락 보완**
   - 주제별 대표 카테고리와 하위 항목 제안
   - 사용자가 쉽게 “추가 탐색이 필요한 빈칸”을 식별할 수 있는 표현(예: ‘추가 고려’ 가지)
3. **접근성**
   - 웹에서 로그인 없이 체험 또는 최소한의 온보딩(정책은 제품 결정 사항)
4. **결과물 활용**
   - 공유/내보내기로 문서·프레젠테이션·기획 산출물에 쉽게 전환

## 9) 사용자 경험(UX) 개요
- 홈: 주제 입력 → (선택) 목적/대상/깊이 설정
- 생성: 로딩/진행 상태 표시 → 결과 마인드맵 렌더링
- 편집: 핵심 가지 다듬기(추가/삭제/정렬)
- 저장/공유: 링크 공유 또는 이미지 내보내기

## 10) 비기능 요구사항(NFR)
- 성능: 사용자가 기다릴 수 있는 수준으로 빠른 1차 결과 제공(“초기 초안 우선”)
- 신뢰/안전: 사실 단정/전문 자문으로 오해되지 않도록 안내 및 표현 관리
- 개인정보/보안: 사용자가 입력하는 업무/학습 내용이 민감할 수 있으므로 보관/삭제/공유 범위에 대한 명확한 정책 필요

## 11) 성공 지표(Success Metrics)
- 마인드맵 “완성까지 걸린 시간”의 유의미한 감소(자기보고/측정 이벤트 기반)
- 생성 후 편집/저장/내보내기 전환율
- 재방문율(주간/월간) 및 반복 생성 횟수

## 12) 리스크 / 가정 / 오픈 이슈
### 리스크
- AI 생성 품질 편차(주제에 따라 과도한 일반화/누락/과잉 세분화)
- 사용자가 기대하는 “정답”과 “아이디어 초안”의 불일치

### 가정
- 사용자는 완성본보다 “빠르게 시작할 수 있는 구조화된 초안”에 가치를 둔다.

### 오픈 이슈(결정 필요)
- 로그인/저장 정책(비회원 체험 범위, 저장 기간)
- 공유 방식(공개 링크/비공개 링크/권한)
- 내보내기 형식 우선순위(이미지 vs PDF vs 기타)
